# Misaka Changelog

## v1.0.0-alpha.1 (2026-02-26 to 2026-03-07)

> 首个版本工作明细。该文档用于完整记录首个版本开发周期内的功能交付、体验优化、问题修复与发布准备事项，适合作为工作量统计、阶段复盘与绩效佐证材料。

### 统计口径
- 统计范围：仓库首个提交至 2026-03-07 的全部主线开发记录
- 处理原则：按模块归类，合并重复提交，将技术描述转换为用户价值表述
- 已忽略项：纯 merge 提交、极少量纯维护性噪音项

## 1. 核心产品定位与基础能力

### 新功能
- 完成 Misaka 首个可用版本搭建，提供基于 Python + Flet 的 Claude Code 桌面图形客户端。
- 建立桌面端产品基础框架，覆盖应用入口、状态管理、服务层、数据库与界面层主流程。
- 支持原生桌面窗口运行，区别于浏览器页面式封装工具，提供更完整的桌面端使用体验。
- 建立与 Claude Code CLI 的集成基础，使桌面界面能够承载实际对话与工具调用能力。

### 改进
- 持续梳理并优化服务层、数据库层与整体架构组织，为后续功能扩展提供稳定基础。
- 补充并完善架构文档、项目说明和开发指引，降低后续维护和协作成本。

## 2. 聊天能力与对话体验

### 新功能
- 支持多轮对话能力，用户可在桌面端持续与 Claude Code 进行上下文连续交流。
- 支持流式响应显示，模型内容可逐步输出，提升等待过程中的可感知反馈。
- 新增消息重新生成功能，便于用户对当前回答不满意时快速获取替代结果。
- 新增回复复制功能，便于将模型输出快速复用到编辑器、文档或其他工具中。
- 支持在聊天中插入文件路径，帮助用户更高效地引用本地项目内容。
- 新增图片附件支持，使聊天输入内容从纯文本扩展到图像上下文。
- 持续增强附件能力，支持更丰富的聊天输入材料组织方式。
- 支持在聊天消息中展示更丰富的内容类型与交互形式，提升复杂任务场景下的信息承载能力。

### 改进
- 优化 StreamingMessage、消息列表与消息项渲染逻辑，提升长回复与连续更新时的稳定性。
- 改进消息列表流式同步行为，减少生成过程中消息跳动、错位或状态不同步问题。
- 提升聊天组件缓存与增量更新能力，减少频繁整页刷新带来的性能损耗。
- 优化消息处理流程，使消息展示、状态刷新和交互反馈更流畅。
- 改进消息项布局与视觉结构，提升文本、图片与混合内容的可读性。
- 优化聊天组件交互一致性，让复制、查看、定位、展开等操作体验更加统一。
- 优化背景流式任务管理，提升长时间生成、复杂对话和多状态切换场景下的可控性。
- 改进聊天模块中的上下文菜单体验，提升常用操作触达效率。
- 提升聊天页面空状态和欢迎态的表达清晰度，让首次进入与无会话场景更易理解。
- 精简 ChatView 层级与头部结构，使用户注意力更聚焦于对话内容本身。
- 对聊天页面布局和样式进行多轮打磨，增强主工作区的空间利用率和视觉秩序。

### 修复
- 修复 ClaudeService 中异步生成器清理问题，减少流式会话结束后的异常残留状态。
- 修复消息列表与流式更新相关的同步细节问题，提升对话展示稳定性。
- 修复部分聊天界面布局不一致问题，提升不同消息内容下的显示统一性。
