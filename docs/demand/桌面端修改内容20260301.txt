整个工程有些bug和问题需要排查修复，以及有一些需要新增的功能需要增加，具体如下：
1. 在聊天模块中，对一些功能进行重构：
   - 对话类型的选择同样使用下拉框的形式
   - 去掉聊天模块中的模型选择下拉框，并在底部用户输入框中，输入 / 后展示的命令添加一个 /model 命令，用于模型选择，具体为，选择了 /model 之后会继续弹出模型选择框（模型选择框展示形式与输入 / 后打开相关命令的显示框展现形式相同），四种模型配置选择，默认分别为：Default、Sonnet、Opus、Haiku。实际上这些模型显示都是通过读取 claude code 中全局 settings.json 配置文件来进行显示，在 settings.json 中对应的配置参数为 env 中的如下参数配置：ANTHROPIC_MODEL(Default)、ANTHROPIC_DEFAULT_OPUS_MODEL(Opus)、ANTHROPIC_DEFAULT_HAIKU_MODEL(Haiku)、ANTHROPIC_DEFAULT_SONNET_MODEL(Sonnet)。若配置文件中对应参数有进行配置，则读取对应配置模型名称展示，否则显示默认名称。默认情况下选中 Default 模型。在用户进行性对话时，给 claude-agent-sdk 传递的模型类型即为选中的模型。
  - 在聊天模块下方对话框部分，附件上传图标右边新增一个命令图标，点击该图标的功能与在对话框中输入 / 后的功能完全一致。目的是针对普通用户在不知道输入 / 可展示可用命令的情况下，直接点击也可显示可用命令列表。
  - 检查聊天模块的实现，是否正确调用了 claude-agent-sdk 进行对话，检查当 claude code 调用了 skill、agent、mcp时在聊天窗口中是否也会实时展示。并检查当前聊天窗口中会话的展示是否正确，我发现会话展示上有一些问题，有些 claude 或者 You 没有内容也被作为一条信息展示出来了，排查相关问题。
  - 当用户进行对话时，一些提示文字例如 Thinking 这些文字，可以有一些文字颜色上的波浪显示动画。
2. 设置模块中：
  - 去掉 claude code 卡片
  - Claude CLI 设置 这个卡片名称修改为 Claude Code Router。并且重写这个功能。将该卡片展现为设置列表形式，支持用户设置多个 settings 配置。点击新增可以添加配置数据，新增配置的表单页面元素包括：供应商名称、API Key、请求地址、主模型、Haiku模型、Opus模型、Sonnet模型、是否为 Agent Team 模式、配置 JSON。其中配置 JSON 新增会有默认 JSON 数据，即为读取 ~/.claude/settings.json 中的内容，但其中 env 配置为 {}（空配置 json）。其中若修改主模型、Haiku模型、Opus模型、Sonnet模型、是否为 Agent Team 模式这五个表单项，配置 JSON 中的对应配置也会对应进行修改，他们是双向绑定联动的关系。模型配置对应关系参考前面第1个功能点中的内容，是否为 Agent Team 配置项对应为 CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS，若选中该表单项，该值为"1"，取消选中该配置项从 JSON 中删除。注意在双向绑定时当配置 JSON 中没有相关参数配置需要自动加上配置，模型表单项为空时配置 JSON 也要将对应的配置项删除，不能直接置为空字符串。新增完配置之后只是保存了当前配置，展示在列表中，并没有真正启用。点击该条设置列表上的启用按钮，会将其中保存的配置 JSON 直接替换掉 ~/.claude/settings.json 中的内容，同时该列表按钮为不可用状态，显示为使用中，这才真正启用了该配置。另外配置列表支持增删改操作。在默认情况下，配置列表中会有一条 Default 配置数据的，即直接读取 ~/.claude/settings.json的数据并为启用状态，在系统第一次打开时检查是否保存了该默认配置，未保存则进行默认配置初始化操作。
